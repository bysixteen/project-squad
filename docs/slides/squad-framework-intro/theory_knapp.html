Built on Jake Knapp's Design Sprint methodology

Refactoring the classic 5-day process for the AI era
sprint_classic.ts Reference: Sprint (Knapp, 2016)
class DesignSprint {
constructor() {
this.team = "7 Humans in a Room";
this.duration = "5 Days (Mon-Fri)";
this.devices = false; // No distractions
}
execute() {
Monday("Map");
Tuesday("Sketch");
Wednesday("Decide");
Thursday("Prototype");
Friday("Test");
}
}
sprint_squad.ts v1.1.0 Adaptation
class SquadSprint extends DesignSprint {
constructor() {
super();
// Replace meeting room with context
this.team = "7 AI Personas";
this.duration = "Async / Instant";
}
execute() {
// Same phases, executed by AI
const context = loadContext();
Map(context);
Sketch(context.personas);
Decide(context.dissent);
Synthesise(); // Replaces Test
}
}
Why Synthesise?
We replace "Friday Testing" with "Synthesis" because we are building the spec, not the final product. Validation happens in the PR.
